# 🎧 Audio Splitter CLI

Быстрая и экономичная утилита для нарезки аудиофайлов из командной строки.  
Использует **нативный FFmpeg-движок** для потоковой обработки, что гарантирует потребление **~20–50 МБ ОЗУ** независимо от длины файла. Идеально подходит для аудиокниг, подкастов, лекций и длинных записей.

## ✨ Возможности

- 🧠 **Умная нарезка по времени**: ищет ближайшую паузу в заданном окне, чтобы не обрывать слова
- ⚡ **Потоковая обработка**: без загрузки файла в RAM, в 5–10 раз быстрее `pydub`-аналогов
- 🔇 **Три режима**: по длительности, по тишине, по таймкодам
- 📊 **Прогресс и метрики**: `tqdm`-бар, ETA, скорость и точный замер времени выполнения
- 📁 **Умные пути**: автоматическое создание папки `<имя_файла>/` рядом с исходником
- 🐍 **Современный стек**: `uv`, `click`, `tqdm`, PEP 621/735, полная типизация

## 📦 Требования

- Python 3.10+
- **FFmpeg** (системная установка, обязателен)
- `uv` (рекомендуется для установки и управления зависимостями)

### Установка FFmpeg
```bash
# Ubuntu/Debian
sudo apt update && sudo apt install -y ffmpeg

# macOS
brew install ffmpeg

# Windows
winget install FFmpeg
```
Проверка: `ffmpeg -version`

## 📖 Использование

### Базовый запуск
```bash
audio-splitter audiobook.mp3
```
**Что произойдёт по умолчанию:**
- Нарезка каждые `15 мин` (900 сек)
- В окне `±15 сек` от каждой точки алгоритм найдёт ближайшую паузу
- Результат сохранится в папку `audiobook/` рядом с файлом
- Формат вывода совпадёт с исходным

### Примеры команд
```bash
# Жёсткая нарезка ровно по таймеру (без поиска пауз)
audio-splitter lecture.wav --chunk 600 --no-smart

# Нарезка по тишине
audio-splitter podcast.mp3 --mode silence --min-silence 800 --thresh -35

# Кастомная папка, формат и окно поиска
audio-splitter meeting.mp3 --out parts/ --format flac --tolerance 20

# Режим таймкодов (в разработке)
audio-splitter interview.mp3 --mode timestamps
```

## 🎛 Справка по параметрам

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `input_file` | Путь к аудиофайлу | *(обязательный)* |
| `--mode` | Режим: `duration`, `silence`, `timestamps` | `duration` |
| `--chunk` | Длительность чанка (сек) | `900` (15 мин) |
| `--smart / --no-smart` | Искать тишину рядом с целевой точкой | `True` |
| `--tolerance` | Окно поиска тишины ±сек | `15.0` |
| `--min-silence` | Мин. длина паузы (мс) | `1000` |
| `--thresh` | Порог тишины (dBFS) | `-40` |
| `--out` | Папка для результатов | `<имя_файла>/` рядом с исходником |
| `--format` | Формат вывода | Авто (как у входа) |

Полная справка: `audio-splitter --help`

## ⚙️ Как это работает

| Компонент | Реализация |
|-----------|------------|
| **Движок** | Прямые вызовы `ffmpeg`/`ffprobe` через `subprocess`. Аудио никогда не загружается в Python. |
| **Память** | Потоковая обработка. Потребление ~20–50 МБ независимо от длины файла. |
| **Скорость** | При совпадении форматов используется `-c copy` (мгновенное копирование фреймов). |
| **Поиск тишины** | FFmpeg-фильтр `silencedetect`. Вывод парсится из `stderr`, точки выбираются по ближайшей середине паузы. |
| **Точность реза** | Frame-accurate. Для MP3 возможен сдвиг ±1 фрейма (~26 мс), что для речи незаметно. Для sample-accurate используйте `--format wav`. |

## 🛠 Устранение проблем

| Симптом | Решение |
|---------|---------|
| `ffmpeg: command not found` | Установите FFmpeg (см. раздел «Требования») и убедитесь, что он в `PATH` |
| Процесс завершается с `Убито / Killed` | В этой версии решено архитектурно. Если повторяется → проверьте битый файл или используйте `--no-smart` |
| Неточные границы чанков в MP3 | Особенность `-c copy`. Для точности до сэмпла добавьте `--format wav` или `--format flac` |
| Пути с пробелами/кириллицей ломаются | Не должны: `subprocess.run` со списком аргументов экранирует их автоматически. Обновите `uv tool install --force .` |
